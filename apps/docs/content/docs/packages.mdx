The repo is easier to understand when each package has a sharp responsibility.

## Package map

| Package | Responsibility |
| --- | --- |
| `uniwind-router` | Platform-aware TanStack Router bridge with a native-friendly `Link` and history helpers |
| `uniwind-router-tools` | Route generation and watch tooling for the shell apps |
| `ui` | Shared React Native Reusables + Uniwind surface for primitives, recipes, and blocks |
| `todo-universal` | Shared route modules, store, views, and styles for the universal todo demo |

## `uniwind-router`

This package is the key interoperability layer. The important idea is simple:

- Web can keep browser-native link behavior.
- Native can map navigation onto `Pressable` and memory history.
- The rest of TanStack Router stays shared.

## `ui`

`ui` now owns the shared UI surface the repo actually uses:

- repo-owned React Native Reusables style components
- Uniwind utilities
- native primitives without import collisions
- shared composition blocks used in docs and app flows

## `todo-universal`

This is the package the docs spend the most time on because it is where the real product demo lives:

- route definitions
- route search parsing
- route loaders and route context helpers
- todo store and stateful feature logic
- view components for list, detail, and stats flows
